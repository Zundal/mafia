# 게임 플레이 HUD 고도화 설계

- **작성일**: 2026-06-28
- **범위**: 순수 UI/UX (게임 규칙·상태머신·API 액션 불변)
- **초점**: 밤/낮 페이즈의 게임 플레이 경험

## 1. 배경 / 문제

`app/game/page.tsx`의 밤(`:418~`)·낮(`:593~`) 페이즈는 3D 아파트(`GameCanvas`)를 전체 화면 배경으로 두지만, 하단 액션 패널이 `max-h-[54vh]`(밤)/`max-h-[48vh]`(낮)을 차지하고 상단에 `PhaseIndicator`+`PhaseTimer` 바까지 겹쳐 **3D 월드가 실질적으로 20~30vh만 보인다.** "폰 한 대를 돌려가며 아파트를 돌아다닌다"는 게임 콘셉트가 약화된다.

부차 문제:
- 밤 대상 선택 버튼(`:504` grid-cols-2, `p-3`)의 터치 영역이 작다.
- 경찰 조사 결과는 패널이 펼쳐졌을 때만 보인다(패널을 접으면 사라짐).
- 낮 페이즈가 "토론 시간"이라는 점이 카피로 드러나지 않는다.
- 밤 종료 비활성 버튼(`:573`)의 대비가 낮다.

## 2. 목표 / 비목표

**목표**
- 밤/낮 페이즈에서 플레이어가 3D 월드를 크게 보고 자유롭게 이동·대화할 수 있게 한다.
- 필요할 때만 액션 패널을 펼쳐 상호작용한다.
- 모바일 터치 사용성과 정보 가독성을 끌어올린다.

**비목표 (이번 범위 아님)**
- 게임 규칙/승패/페이즈 전이 변경, 신규 액션·페이즈 신설.
- `lib/game-logic.ts`, `app/api/game/route.ts` 로직 변경.
- 데모/스토리/홈 화면 개편(별도 작업).

## 3. 설계

### 3.1 핵심: `CollapsiblePanel` (신규 공용 컴포넌트)

`app/components/CollapsiblePanel.tsx` 신설. 밤·낮 하단 패널의 공통 크롬(panel-dark, `rounded-t-3xl`, 드래그 핸들, 스크롤)을 한 곳으로 모아 접힘/펼침을 담당한다.

**Props**
- `peek: ReactNode` — 접힘 상태에서 보이는 한 줄 요약(아이콘+텍스트, 탭하면 펼침).
- `children: ReactNode` — 펼침 상태 본문.
- `maxHeightClass?: string` — 펼침 높이 (기본 `max-h-[54vh]`).
- `defaultExpanded?: boolean` — 초기 상태(기본 `true`).

**동작**
- 내부 `useState<boolean> expanded`. 상단 핸들 영역이 토글 버튼(역할 `button`, `aria-expanded`).
- **펼침**: 기존과 동일한 스크롤 패널 + 핸들. 핸들 탭 시 접힘.
- **접힘**: 패널이 아래로 내려가 `peek` 바(고정 높이 ~56px)만 남고, 위로 향한 ⌃(chevron)으로 펼침 유도. 탭하면 펼침.
- transition: `transform`/`max-height` 기반의 부드러운 애니메이션(`transition-transform duration-300`), 기존 globals.css 토큰/관례 준수.
- `pb-safe`(노치 대응) 유지.

> 접힘/펼침은 **로컬 UI 상태**이며 서버 게임 상태와 무관하다. 폴링·페이즈 전환 시 리셋되지 않도록 페이즈 키가 아닌 컴포넌트 마운트 기준으로 유지한다.

### 3.2 밤 페이즈 적용 (`:439~589`)

- 기존 하단 `div.panel-dark`를 `CollapsiblePanel`로 교체. 본문(미션/능력 선택/준비 현황/밤 종료)은 그대로 children으로 이동.
- `peek` 내용은 현재 플레이어 상태에 따라:
  - 미행동 능력자: `🍷 제거 대상을 선택하세요` / `🕵️ 조사할 사람을 고르세요` / `🧹 보호할 사람을 고르세요`.
  - 행동 완료(경찰): `🕵️ 조사 결과 — 🚨 범인입니다! / ✅ 범인이 아닙니다` (조사 결과 상시 노출).
  - 행동 완료(기타): `✓ 완료 · 다른 사람 대기 중 (readyCount/total)`.
  - 비능력자: `😴 잠든 척 · 지도를 자유롭게 이동하세요`.
- **터치 영역 확대**: 대상 선택 버튼 `p-3` → `p-4`, `min-height` 부여(예: `min-h-[52px]`).
- **밤 종료 버튼 가독성**: 비활성 시 배경/테두리 대비 상향(현 `disabled:opacity-40` 대신 명시적 비활성 스타일 유지·강화).

### 3.3 낮 페이즈 적용 (`:593~685`)

- 동일하게 `CollapsiblePanel`로 교체. 본문(아침 뉴스/생존자/투표 시작)은 children.
- `peek`: `🗣️ 토론 시간 · 🗳️ 준비되면 투표 시작`.
- **토론 프레이밍**: 패널 본문 상단에 "🗣️ 토론 시간 — 자유롭게 이야기하고 범인을 추리하세요" 소제목 추가(카피·시각만, 규칙 변경 없음). 기존 "지도를 돌아다니며..." 문구와 통합.

### 3.4 비적용 페이즈

- setup/voting/ended/참여 화면은 전체 스크롤 레이아웃이라 `CollapsiblePanel` 미적용(현행 유지).

## 4. 컴포넌트 경계 / 영향 파일

- 신규: `app/components/CollapsiblePanel.tsx` (자기완결, 게임 로직 의존 없음 — peek/children만 받는다).
- 수정: `app/game/page.tsx` 밤·낮 블록(JSX 교체 + peek 계산), 터치/가독성 미세 조정.
- 무변경: `lib/*`, `app/api/*`, 기타 컴포넌트.

## 5. 검증

- `bun run lint`, `bun run build` 통과.
- 수동: 데모로는 검증 불가(별도 페이지)이므로 게임 플로우로 밤→낮 진입, 패널 접힘/펼침 시 3D 월드 노출 면적 확대 확인, 모바일 폭(375px)에서 버튼 터치 영역·peek 가독성 확인.
- 회귀: 능력 실행/밤 종료/투표 시작 등 기존 액션이 그대로 동작하는지 확인.

## 6. 리스크

- `fixed inset-0` 컨테이너 안에서 접힘 시 패널이 화면 밖으로 나가며 핸들/peek 접근이 막히지 않도록 z-index·하단 고정 유지.
- 접힘 상태에서도 `MusicPlayer`(우하단 고정)와 겹치지 않도록 여백 확인.
